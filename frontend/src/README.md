# Frontend - Bus Booking System

This folder contains all the **user interface** files for the Bus Booking System.

## Structure

```
frontend/
├── src/
│   ├── templates/          # Thymeleaf HTML templates
│   │   ├── index.html           - User login page
│   │   ├── trip_details.html    - Trip booking form
│   │   ├── trip_history.html    - User trip history page
│   │   └── admin.html           - Admin dashboard
│   └── static/             # Static web resources
│       ├── css/
│       │   ├── style.css        - Main stylesheet
│       │   └── admin.css        - Admin panel styles
│       └── js/
│           ├── script.js        - Common JavaScript functions
│           └── admin.js         - Admin dashboard functionality
└── README.md
```

## Pages Overview

### 1. **index.html** - User Login
- Simple login form
- Accepts username and contact number
- Creates/retrieves user from database
- Links to admin login

### 2. **trip_details.html** - Trip Booking
- Lists available vehicles
- Lists available drivers
- Date range selection (start & end dates)
- Pickup/destination point input
- Booking confirmation

### 3. **trip_history.html** - Trip History
- Displays all user bookings
- Shows trip details (vehicle, driver, dates, route)
- Trip status indicator (booked/cancelled)
- Cancel button for active trips

### 4. **admin.html** - Admin Dashboard
- Admin login form
- Tabbed dashboard for:
  - **Bookings**: View all system trips, cancel any trip
  - **Vehicles**: Add new vehicles, delete existing vehicles
  - **Drivers**: Add new drivers, delete existing drivers

## Technologies

- **Templating**: Thymeleaf (Server-side rendering)
- **Styling**: CSS3 with responsive design
- **Scripting**: Vanilla JavaScript (no dependencies)
- **Backend Integration**: Thymeleaf expressions for dynamic content

## How It Works

1. **Server-side Rendering**: All HTML is generated by Spring Boot using Thymeleaf
2. **Form Submission**: Forms send POST requests to backend endpoints
3. **Dynamic Content**: Thymeleaf loops through database entities and renders them
4. **Client-side Validation**: JavaScript provides real-time form validation
5. **Session Management**: User session is maintained across pages

## JavaScript Features

### script.js
- Date validation (prevents selecting past dates)
- Ensures end date is after start date
- Real-time date range validation

### admin.js
- Tab switching functionality
- Admin dashboard navigation
- Modal confirmations for delete operations

## Styling

The CSS provides:
- Responsive layout (works on desktop and mobile)
- Clean, professional UI
- Color-coded buttons (green for success, red for delete)
- Table styling for data display
- Form styling for user inputs
- Tab interface for admin dashboard

## Integration with Backend

Frontend templates reference backend endpoints:

```
- GET  /                    - Renders index.html
- POST /login              - Form action
- GET  /trip_details       - Renders trip_details.html
- POST /book_trip          - Form action
- GET  /trip_history       - Renders trip_history.html
- POST /cancel_trip/{id}   - Form action
- GET  /admin              - Renders admin login
- POST /admin/login        - Form action
- GET  /admin/dashboard    - Renders admin.html
- POST /admin/* endpoints  - Form actions for admin operations
```

## Development Notes

- All templates use Thymeleaf namespace: `xmlns:th="http://www.thymeleaf.org"`
- CSS is preprocessed and served by Spring Boot static resource handler
- JavaScript is vanilla (no jQuery, React, or Vue required)
- Forms use POST method for security
- CSRF protection is enabled for all POST requests
